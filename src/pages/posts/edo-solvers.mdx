I compare two classics: **<span style="color: blue">ode45 in MATLAB</span>** and **<span style="color: blue">Tsit5 in Julia</span>**. Both are explicit Runge–Kutta methods with adaptive step size and they shine on non-stiff problems (smooth dynamics, no crazy-fast components).

**Example of a non-stiff system:** the van der Pol oscillator when μ is small.

$$
\begin{cases}
\begin{align*}
\frac{dx}{dt} &= y \\
\frac{dy}{dt} &= μ(1 − x^2) y − x
\end{align*}
\end{cases}
$$

## What these solvers actually do 

At each time step they don’t just take one guess; they compute a small bundle of intermediate evaluations (the famous “k’s” in Runge–Kutta). From that bundle they produce **two answers of different orders** (e.g., order 5 and order 4). The **difference between them is an error estimate**. If the error looks big, they shrink the step; if it’s small, they grow it. That’s adaptive stepping.

One more trick you’ll see later is **FSAL (First Same As Last)**: the last function evaluation from a successful step becomes the first evaluation for the next step. It saves one call to `f(t, y)` per step. Tiny detail, real speed in large runs.

## `ode45` (MATLAB)

**TL;DR**: `ode45` uses the **Dormand–Prince 5(4)** pair (7 stages, order 5 with an embedded order-4 estimate). It’s the go-to default for many problems.

Under the hood, it looks like this (conceptually):

$$$
\begin{align*}
y_{n+1} &= y_n + h  Σ b_i  k_i\\
k_i &= f(t_n + c_i h, y_n + h  Σ a_{ij} k_j)  \quad \text{(explicit RK)}
\end{align*}
$$$

Dormand & Prince chose the coefficients to keep the **order-5 solution very accurate** while still getting a good **order-4 error estimate** for step control. `ode45` also uses FSAL, so although the tableau has 7 stages, **you pay 6 evaluations per accepted step**.

### Adaptive step rule

MATLAB controls both **relative** and **absolute** error (`RelTol`, `AbsTol`). A simplified step update looks like:

```matlab
h_next = h * (tol / error)^(1/5)   // 1/5 because main method is order 5
```

In practice, MATLAB dampens that update (safety factors, min/max clamps) to avoid aggressive jumps:

```matlab
h_next ≈ max(h_min, h * max(0.1, 0.8 * (tol/error)^(1/5)))
```

If a step is rejected, it halves the step and retries.

`ode45` also ships a **6th-order dense output** (interpolant), so you can query smooth values between steps—handy for event finding and pretty plots.

### Minimal example: Lotka–Volterra in MATLAB

```matlab
function dydt = lotkavolterra(t,y,alpha,beta,delta,gamma)
    dydt = [y(1)*(alpha - beta*y(2));
            y(2)*(delta*y(1) - gamma)];
end

params = [0.5 0.3 0.2 0.1]; % alpha, beta, delta, gamma
tspan = [0 20];
y0 = [10; 5];
options = odeset('RelTol',1e-6,'AbsTol',1e-9);

[t,y] = ode45(@(t,y) lotkavolterra( ...
    t,y,params(1),params(2),params(3),params(4)), ...
    tspan, y0, options);
```

## `Tsit5` (Julia, DifferentialEquations.jl)

**TL;DR**: `Tsit5` is a **Tsitouras 5/4** pair (2011). It’s explicit RK with an embedded error estimate and FSAL, like DP(5,4), but tuned coefficients make it **a bit more efficient on many problems**. That’s why it’s a popular default in Julia.

Two notes I like:

* The method relaxes some classic tableau constraints, giving more freedom to **minimize truncation error** for the 5th-order solution.
* It includes a **4th-order dense output** for smooth interpolation inside each step.

A common (simplified) step update you’ll see documented is:

```matlab
h_next = 0.9 * (tol / error)^(1/5)
```

Same spirit as `ode45`: safety factor + order-based exponent.

### Minimal example: Lorenz in Julia

```julia
using DifferentialEquations, Plots

function lorenz!(du,u,p,t)
    σ,ρ,β = p
    du[1] = σ*(u[2]-u[1])
    du[2] = u[1]*(ρ-u[3]) - u[2]
    du[3] = u[1]*u[2] - β*u[3]
end

u0    = [1.0, 0.0, 0.0]
p     = (10.0, 28.0, 8/3)
tspan = (0.0, 100.0)

prob = ODEProblem(lorenz!, u0, tspan, p)
sol  = solve(prob, Tsit5(); reltol=1e-8, abstol=1e-8)
```

## Quick comparison

* **Both**: explicit RK, adaptive step, embedded error estimate, FSAL, dense output. Great for **non-stiff** ODEs.
* **`ode45`**: excellent default in MATLAB, very nice **6th-order interpolation**.
* **`Tsit5`**: often **\~10% faster** on standard test sets thanks to coefficient tuning (your mileage may vary).

If I’m in MATLAB, I start with `ode45`. If I’m in Julia, `Tsit5` is the first thing I try.
